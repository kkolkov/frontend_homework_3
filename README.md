Задание
Ожидаемый результат: готова полная оболочка приложения для итогового задания.
Обратите внимание: необязательно создавать все компоненты, некоторых компонентов у вас может не быть, важно соблюдать требования итогового задания.
Описание задания
1. Макет приложения (Layout)
Создайте корневой компонент AppLayout, который задаёт общую структуру страницы:
•	боковая панель (sidebar) фиксированной ширины слева;
•	основная область чата справа (занимает всё оставшееся пространство).
Учитывайте корректное поведение страницы на мобильных устройствах: боковая панель скрывается, появляется кнопка-бургер.
 
2. Sidebar — боковая панель
Компонент Sidebar должен содержать:
•	кнопку «Новый чат» (с иконкой +);
•	поле поиска по чатам (SearchInput);
•	список чатов (ChatList) — здесь необходимо отобразить минимум 5 моковых чатов.
Каждый элемент списка ChatItem должен включать:
•	название чата, обрезанное при переполнении через text-overflow: ellipsis;
•	дату последнего сообщения;
•	кнопки редактирования и удаления (появляются при hover);
•	визуальное выделение активного чата.
 
3. Область чата (ChatWindow)
Компонент ChatWindow (центральная зона) должен включать:
•	заголовок с названием текущего чата и кнопкой настроек;
•	область сообщений MessageList с вертикальной прокруткой;
•	минимум 6 моковых сообщений (3 — от пользователя, 3 — от ИИ-ассистента) с визуальным разделением:
o	сообщения пользователя располагаются справа, имеют отличный от основного цвет фона;
o	сообщения ассистента располагаются слева, имеют аватар или иконку GigaChat.
 
4. Компонент сообщения (Message)
Компонент Message должен:
•	отображать имя или роль отправителя;
•	отображать текст с базовой поддержкой markdown (жирный, курсив, списки, блок кода — через react-markdown);
•	иметь кнопку «Копировать», которая появляется при hover;
•	иметь два визуальных стиля: variant=“user” и variant=“assistant”.
 
5. Индикатор загрузки (TypingIndicator)
Анимированный компонент TypingIndicator (три пульсирующие точки), должен отображаться на месте следующего сообщения ИИ-ассистента во время ожидания ответа пользователем.
Необходимо показывать его в моковом виде постоянно или по prop isVisible.
 
6. Поле ввода (InputArea)
Создайте компонент InputArea в нижней части чата со свойствами:
•	многострочный textarea имеет автоподстройку высоты (до 5 строк);
•	кнопка «Отправить» активна только при непустом вводе;
•	кнопка «Стоп» пока заглушена для будущей функции остановки генерации;
•	отправка по Enter (перенос строки — Shift+Enter) представляет собой только обработчик, без реального запроса;
•	кнопка прикрепления изображения — только иконка, без логики.
 
7. Модальная панель настроек (SettingsPanel)
Компонент SettingsPanel открывается как drawer или modal, должен содержать:
•	возможность выбора модели (<select> с вариантами GigaChat, GigaChat-Plus, GigaChat-Pro, GigaChat-Max);
•	слайдер Temperature (0–2);
•	слайдер Top-P (0–1);
•	числовое поле Max Tokens;
•	Textarea для System Prompt;
•	переключатель темы (светлая или тёмная) — Toggle;
•	кнопки «Сохранить» и «Сбросить».
 
8. Форма авторизации (AuthForm)
Создайте отдельный экран AuthForm, который отображается вместо AppLayout, если пользователь не авторизован. AuthForm должен содержать:
•	поле для ввода Credentials (Base64-строка, тип password);
•	радио-кнопки для выбора Scope: GIGACHAT_API_PERS, GIGACHAT_API_B2B, GIGACHAT_API_CORP;
•	кнопку «Войти»;
•	базовую клиентскую валидацию (поле не должно быть пустым) с выводом ошибки.
 
9. Компоненты ошибок
Создайте компоненты:
•	ErrorMessage — inline-блок с текстом ошибки и иконкой, например, красный баннер под полем ввода;
•	EmptyState — заглушку для пустого чата: иллюстрация или иконка + текст «Начните новый диалог».

Технические требования
•	Обязательно используйте в проекте связку React + TypeScript.
•	Используйте только моковые данные, не делайте никаких реальных запросов к API.
•	Подбирайте инструменты стилизации на своё усмотрение: CSS Modules, Tailwind или Styled Components.
•	Настройте тёмную и светлую темы через CSS-переменные (–color-bg, --color-text и т.д.).
•	Создайте адаптивность: корректное отображение от 320px (mobile) до 1440px (desktop).
•	Проследите, что react-markdown подключён и рендерит моковый markdown-текст в компоненте Message.
Чего на этом этапе делать не нужно
•	Подключать GigaChat API или любые другие запросы.
•	Реализовывать хранение данных (localStorage, IndexedDB).
•	Писать state management (Zustand, Redux).
•	Покрывать код тестами.
