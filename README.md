Это статичная реализация пользовательского интерфейса (UI) для будущего чат-приложения на базе GigaChat. Проект выполнен в рамках самопроверки (Домашнее задание №3) и представляет собой "скелет" приложения с моковыми данными, без подключения к реальному API.

> **Цель работы:** Создать полную оболочку приложения, используя компонентный подход React, TypeScript для типизации и адаптивную верстку с поддержкой темной и светлой темы.

## ✨ Функциональность (Реализованные компоненты)

В проекте созданы все ключевые UI-элементы, необходимые для итогового приложения:

1.  **📐 Макет (AppLayout)**
    - Корневая структура: фиксированная боковая панель слева и адаптивная область чата справа.
    - 📱 **Адаптивность:** На мобильных устройствах панель скрывается, появляется кнопка-бургер.

2.  **⬅️ Боковая панель (Sidebar)**
    - Кнопка «Новый чат».
    - Поле поиска.
    - Список чатов (`ChatList`) с 5 моковыми элементами.
    - Элемент чата (`ChatItem`) c названием (обрезанным через `...`), датой, кнопками редактирования/удаления (появляются при hover) и визуальным выделением активного чата.

3.  **💬 Область чата (ChatWindow)**
    - Заголовок с названием чата и кнопкой настроек.
    - Список сообщений (`MessageList`) с 6 моковыми сообщениями (3 от пользователя, 3 от ассистента).
    - Правильное визуальное позиционирование: сообщения пользователя — справа, ассистента — слева с иконкой.

4.  **🧩 Сообщение (Message)**
    - Отображение роли отправителя.
    - 🎨 **Поддержка Markdown:** Жирный текст, курсив, списки и блоки кода рендерятся через `react-markdown`.
    - Кнопка «Копировать» (появляется при hover).
    - Два варианта стилей: `user` и `assistant`.

5.  **⏳ Индикатор печати (TypingIndicator)**
    - Анимированные три пульсирующие точки. Отображается в моковом режиме, имитируя ожидание ответа от ИИ.

6.  **⌨️ Поле ввода (InputArea)**
    - Многострочное поле (`textarea`) с автоподстройкой высоты (до 5 строк).
    - Кнопка «Отправить» активна только при наличии текста.
    - Отправка по `Enter` (обработчик заглушен, `Shift + Enter` для новой строки).
    - Заглушки для кнопок «Стоп» и «Прикрепить файл».

7.  **⚙️ Панель настроек (SettingsPanel)**
    - Реализована в виде модального окна (drawer/modal).
    - Содержит: выбор модели, слайдеры Temperature и Top-P, поле Max Tokens, `textarea` для System Prompt, переключатель темы (Toggle), кнопки «Сохранить» и «Сбросить».

8.  **🔐 Форма авторизации (AuthForm)**
    - Отдельный экран, имитирующий неавторизованное состояние.
    - Поле для ввода Credentials (тип `password`), радио-кнопки выбора Scope, кнопка «Войти».
    - Реализована базовая клиентская валидация (проверка на пустоту) с выводом ошибки.

9.  **🚨 Компоненты обратной связи**
    - **ErrorMessage:** Инлайн-блок с ошибкой (например, под полем ввода).
    - **EmptyState:** Заглушка для пустого чата (иконка и текст "Начните новый диалог").

✅ Чек-лист самопроверки

Все перечисленные в задании компоненты созданы.
TypeScript: Для каждого компонента прописаны корректные типы props.
Адаптивность: Корректное отображение на экранах от 320px до 1440px (бургер-меню, резиновая верстка).
Темы: Настроены светлая и темная темы через CSS-переменные.
Маркдаун: react-markdown успешно парсит моковый текст в сообщениях.

Хранение данных (LocalStorage, IndexedDB).
